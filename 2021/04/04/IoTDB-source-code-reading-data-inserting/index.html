IoTDB源码阅读（一）：数据插入

发表于 2021-04-04 | 分类于 IoTDB ， 源代码阅读
字数统计: 1231 字 | 阅读时长 ≈ 5 分钟

Insert 单行数据

单行数据指一个时间戳下，一个设备多个测点的一行测量值。
下面以 Session 的 insertRecord 接口为例，展示插入数据的过程。

简单来说，单行数据的插入分成三个阶段。分别是插入计划生成、插入计划路由、数据插入。

插入计划生成

在 TSServiceImpl.java 中，根据收到的插入参数，生成对应的 InsertRowPlan。在检查完用户状态、执行权限、数据库状态等信息后，将该 InsertRowPlan 传给 PlanExecutor。

插入计划路由

PlanExecutor 的 processNonQuery 接收的参数类型为 PhysicalPlan，包含删除、插入、批量插入等类型，这里我们传入的是插入类型的计划，于是将其路由到 insert 函数中。在 PlanExecutor 的 insert 函数中，其调用 StroageEngine 的 insert 接口来完成插入。

插入

StorageEngine 找到该 InsertRowPlan 对应的 StorageGroup 的 StorageGroupProcessor，然后继续调用其 insert 接口。

StorageGroupProcessor 找到对应的 TsFileProcessor，继续调用其 insert 接口。

TsFileProcessor 中维护了一个 MemTable，其将数据保存在内存中。在接收到插入的计划后，TsFileProcessor 将这些数据插入到 MemTable 中。

在 MemTable 中，其使用的是一个 TVList 数据结构来存储插入的数据，该数据结构每次插入时都将一个 <TimeStamp, Value> 对保存下来。

插入多行数据

Session 的 insertTablet 可以插入多行数据，其大致顺序跟插入单行数据类似。区别在于，生成的插入计划不是 InsertRowPlan，而是 InsertTabletPlan。在 PlanExecutor 中路由时，会路由到 insertTablet 函数中。在插入过程中，各个层级的 Processor 调用的是 insertTablet 函数而不是 insert 函数。在 StroageGroupProcessor 中，其会将一个 Tablet 中的数据分成多个小批次，然后将这些分批的数据分别插入到 TsFileProcessor 中。
